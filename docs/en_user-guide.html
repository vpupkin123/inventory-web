1. Introduction

Inventory Web is a web application for tracking computer equipment in an organization. The system allows you to:

  • Upload computer data from JSON reports collected on client machines
  • Automatically create employee accounts
  • Assign computers to specific users or store them in a virtual warehouse
  • Maintain a complete history of all equipment movements
  • Export data to Excel for reporting purposes
💡 Note: The application supports two interface languages — Russian and English. You can switch languages at any time via the menu in the top right corner.

2. Login

1 Open the system URL in your browser (e.g., http://inventory-web/)
2 Enter your login and password
3 Click the "Login" button
️ Important: On your first login, the system will prompt you to change your temporary password. This is a mandatory security step.

3. Change Password

If the system redirected you to the password change page:

1 Enter a new password (minimum 6 characters)
2 Repeat the password for confirmation
3 Click "Change Password"
✅ Tip: Use a strong password containing uppercase and lowercase letters, numbers, and special characters.

4. Language Switching

In the top right corner of the navigation bar, there is a dropdown menu with flags:

  • 🇬🇧 EN — English
  • 🇷🇺 RU — Russian

The selected language is saved for your account and will be used on subsequent logins.

5. Dashboard

After logging in, you'll be taken to the dashboard, which contains:

📊 Statistics

  • Total Computers — total number of PCs in the database
  • Assigned — computers assigned to employees
  • On Warehouse — available computers awaiting assignment
  • Unprocessed — new reports requiring processing
  • Total Users — number of user accounts in the system

⚡ Quick Actions

Buttons for the most common operations:

  • Process Queue — go to report processing (appears if there are unprocessed items)
  • Upload Report — upload a new JSON file
  • View Computers — open the equipment registry
  • Manage Users — available only to administrators

Recent Activity

Two tables show the last 5 report uploads and the last 5 computer transfers.

6. Uploading Reports

JSON reports are generated by the client application on employee computers and contain complete information about hardware, OS, and network.

How to upload a report:

1 Navigate to "Reports" in the top menu
2 Click the "Upload new report" button
3 Select the JSON file on your computer
4 Click "Upload"
💡 Duplicate Protection: The system automatically calculates the file hash and prevents uploading the same report twice. If you try to upload an existing file, you'll see an appropriate message.

What happens after upload:

  • The file is saved in the system
  • Data is extracted and filtered (USB devices are ignored)
  • The computer is added to the processing queue with "New" status
  • The computer is temporarily assigned to the virtual warehouse

7. Processing Queue

This is the key section for administrators. Here, new computers from reports are processed.

Actions with computers in the queue:

  • Select with checkboxes the computers you want to process (or use "Select All")
  • Add a comment for each PC (e.g., "Half-dead PC", "Ivanov's PC")
  • Check "Create New User" if you need to create an account for the PC owner
  • If the checkbox is checked — a "Settings" button will appear

Configuring a new user:

1 Click the "Settings" button
2 In the opened window, correct the full name (e.g., remove the word "Warehouse")
3 The login will be auto-generated according to GOST standard (e.g., ivanov.ii)
4 If the login is already taken — the system will automatically select a unique variant
5 Click "Save"
️ Important: You cannot click "Process Selected" until all users with "Create New User" checked have been configured.

After clicking "Process Selected":

  • For PCs with "Create New User" checked — a user account is created, and the PC is assigned to the new user
  • For other PCs — they remain on the warehouse but are marked as "Processed"

8. Computer Registry

The "Computers" section shows all PCs ever uploaded to the system.

The table contains:

  • ID — unique number
  • PC Name — network computer name
  • Current Owner — user login or "Warehouse"
  • Status — "New" or "Processed"
  • Actions — "View" and "Transfer" buttons
✅ Export: In the top right corner, there's an "Export to Excel" button to download the entire registry.

9. Computer Card

Detailed information about a specific PC, including:

  • Serial number
  • Motherboard
  • Processor (model, cores, threads)
  • RAM (total size and modules)
  • Storage devices (excluding USB devices)
  • Operating system
  • IP address
  • Current owner
  • Reported by (who submitted the report)
  • Comment
  • Date added to database

From the card, you can proceed to transfer the computer or return to the list.

10. Transfer Computer

Through this form, you can change the computer's owner.

Possible scenarios:

  • From Warehouse → Employee: select a user from the list
  • From Employee → Employee: select another user
  • From Employee → Warehouse: check "Return to Warehouse"
1 The "From" field shows the current owner
2 Select the new owner from the dropdown list
3 Add a comment to the transfer if necessary
4 Click "Confirm Transfer"
💡 History: Every transfer is automatically logged in the history journal.

11. Transfer History

The "History" section contains a log of all equipment movements in the system.

Each record displays:

  • Date and time of transfer
  • Computer (name and serial number)
  • From (previous owner or "Warehouse")
  • To (new owner or "Warehouse")
  • Performed by (administrator who executed the transfer)
  • Comment
✅ Export: History can also be exported to Excel.

12. User Management

⚠️ Available only to administrators!

Capabilities:

  • Create user: enter full name — login will be auto-generated according to GOST R 7.0.34-2014
  • Edit: change full name, role, password
  • View PCs: "Computers" button shows assigned equipment and allows adding/removing computers
  • Delete: remove user (only if no computers are assigned to them)

User Roles:

Role Permissions
Administrator Full access to all functions, including user management
Editor Upload reports, processing, computer transfers
Viewer View-only access
Blocked No system access
💡 Auto-login generation: If a login already exists, the system automatically adds additional letters from the first/middle name. For example: ivanov.ii → ivanov.iiи → ivanov.ivan etc.

13. Administrative Scripts

The admin-scripts/ folder contains two helper scripts for administrators:

🔧 fix-permissions.sh — Fix File Permissions

Used after each git pull update to restore correct file permissions.

Usage:

cd /volume1/web/inventory-web
./admin-scripts/fix-permissions.sh

Parameters (optional):

./admin-scripts/fix-permissions.sh [PROJECT_PATH] [OWNER] [GROUP] [WEB_USER]

Example with custom parameters:

./admin-scripts/fix-permissions.sh /volume1/web/inventory-web myuser mygroup www-data
💡 Defaults: The script automatically uses the current user and group, and http user for the web server.

🗑️ reset-inventory.sh — Reset Data

Used to clean the database from test data. Deletes all computers, reports, and transfers, but preserves users.

Usage:

./admin-scripts/reset-inventory.sh

Parameters (optional):

./admin-scripts/reset-inventory.sh [PROJECT_PATH]
⚠️ Caution: The script will ask for confirmation before deleting data.

14. Excel Export

Export functionality is available in the sections:

  • Computers — complete equipment registry
  • History — log of all transfers

How to export:

1 Navigate to the desired section
2 Click the green "Export to Excel" button
3 The file will automatically download in .xls format
4 Open the file in Microsoft Excel, LibreOffice Calc, or another spreadsheet editor
✅ Compatibility: Files correctly display Cyrillic characters and can be saved to .xlsx format through Excel itself.
