1. Введение

Inventory Web — это веб-приложение для учёта компьютерного оборудования в организации. Система позволяет:

  • Загружать данные о компьютерах из JSON-отчётов, собранных на клиентских машинах
  • Автоматически создавать учётные записи сотрудников
  • Закреплять компьютеры за конкретными пользователями или хранить их на виртуальном складе
  • Вести историю всех перемещений оборудования
  • Экспортировать данные в Excel для отчётности
Примечание: Приложение поддерживает два языка интерфейса — русский и английский. Переключение доступно в любой момент через меню в правом верхнем углу.

2. Вход в систему

1 Откройте в браузере адрес системы (например, http://inventory-web/)
2 Введите ваш логин и пароль
3 Нажмите кнопку "Войти"
️ Важно: При первом входе система попросит вас сменить временный пароль. Это обязательный шаг для обеспечения безопасности.

3. Смена пароля

Если система перенаправила вас на страницу смены пароля:

1 Введите новый пароль (минимум 6 символов)
2 Повторите пароль для подтверждения
3 Нажмите "Сменить пароль"
✅ Совет: Используйте надёжный пароль, содержащий буквы разного регистра, цифры и специальные символы.

4. Переключение языка

В правом верхнем углу навигационной панели находится выпадающий список с флагами:

  • 🇬🇧 EN — английский язык
  • 🇷🇺 RU — русский язык

Выбранный язык сохраняется для вашей учётной записи и будет использоваться при следующих входах.

5. Главная страница (Dashboard)

После входа вы попадаете на главную страницу, которая содержит:

📊 Статистика

  • Всего компьютеров — общее количество ПК в базе
  • Назначено — компьютеры, закреплённые за сотрудниками
  • На складе — свободные компьютеры, ожидающие назначения
  • Не обработано — новые отчёты, требующие обработки
  • Пользователей — количество учётных записей в системе

⚡ Быстрые действия

Кнопки для самых частых операций:

  • Обработать очередь — перейти к обработке новых отчётов (появляется, если есть необработанные)
  • Загрузить отчёт — загрузить новый JSON-файл
  • Просмотр компьютеров — открыть реестр оборудования
  • Управление пользователями — доступно только администраторам

📈 Последняя активность

6. Загрузка отчётов

JSON-отчёты формируются клиентским приложением на компьютерах сотрудников и содержат полную информацию о железе, ОС и сети.

Как загрузить отчёт:

1 Перейдите в раздел "Отчёты" в верхнем меню
2 Нажмите кнопку "Загрузить новый отчёт"
3 Выберите JSON-файл на вашем компьютере
4 Нажмите "Загрузить"
💡 Защита от дубликатов: Система автоматически вычисляет хеш файла и не позволит загрузить один и тот же отчёт дважды. Если вы попытаетесь загрузить уже существующий файл, увидите соответствующее сообщение.

Что происходит после загрузки:

  • Файл сохраняется в системе
  • Данные извлекаются и фильтруются (USB-устройства игнорируются)
  • Компьютер добавляется в очередь обработки со статусом "Новый"
  • Компьютер временно закрепляется за виртуальным складом

7. Очередь обработки

Это ключевой раздел для администратора. Здесь обрабатываются новые компьютеры, поступившие из отчётов.

Действия с компьютерами в очереди:

  • Выберите галочками компьютеры, которые хотите обработать (или используйте "Выделить все")
  • Добавьте комментарий к каждому ПК (например, "Полудохлый ПК", "ПК Иванова")
  • Поставьте галочку "Создать пользователя", если нужно создать учётную запись для владельца ПК
  • Если галочка установлена — появится кнопка "Настройка"

Настройка нового пользователя:

1 Нажмите кнопку "Настройка"
2 В открывшемся окне исправьте ФИО (например, уберите слово "Склад")
3 Логин сгенерируется автоматически по ГОСТ (например, ivanov.ii)
4 Если логин уже занят — система автоматически подберёт уникальный вариант
5 Нажмите "Сохранить"
⚠️ Важно: Нельзя нажать "Обработать выбранные", пока все пользователи с галочкой "Создать пользователя" не будут настроены.

После нажатия "Обработать выбранные":

  • Для ПК с галочкой "Создать пользователя" — создаётся учётная запись, ПК закрепляется за новым пользователем
  • Для остальных ПК — они остаются на складе, но помечаются как "Обработанные"

8. Реестр компьютеров

Раздел "Компьютеры" показывает все ПК, когда-либо загруженные в систему.

Таблица содержит:

  • ID — уникальный номер
  • Имя ПК — сетевое имя компьютера
  • Текущий владелец — логин пользователя или "Склад"
  • Статус — "Новый" или "Обработан"
  • Действия — кнопки "Просмотр" и "Передать"
✅ Экспорт: В правом верхнем углу есть кнопка "Экспорт в Excel" для выгрузки всего реестра.

9. Карточка компьютера

Подробная информация о конкретном ПК, содержащая:

  • Серийный номер
  • Материнская плата
  • Процессор (модель, ядра, потоки)
  • Оперативная память (объём и модули)
  • Накопители (без USB-устройств)
  • Операционная система
  • IP-адрес
  • Текущий владелец
  • От кого получен отчёт
  • Комментарий
  • Дата добавления в БД

Из карточки можно перейти к передаче компьютера или вернуться к списку.

10. Передача компьютера

Через эту форму можно изменить владельца компьютера.

Возможные сценарии:

  • Со склада → Сотруднику: выберите пользователя из списка
  • От сотрудника → Сотруднику: выберите другого пользователя
  • От сотрудника → На склад: поставьте галочку "Вернуть на склад"
1 В поле "От кого" отображается текущий владелец
2 Выберите нового владельца из выпадающего списка
3 При необходимости добавьте комментарий к передаче
4 Нажмите "Подтвердить передачу"
💡 История: Каждая передача автоматически записывается в журнал истории.

11. История передач

Раздел "История" содержит журнал всех перемещений оборудования в системе.

Для каждой записи отображается:

  • Дата и время передачи
  • Компьютер (имя и серийный номер)
  • От кого (предыдущий владелец или "Склад")
  • Кому (новый владелец или "Склад")
  • Кто выполнил (администратор, осуществивший передачу)
  • Комментарий
✅ Экспорт: История также может быть выгружена в Excel.

12. Управление пользователями

️ Доступно только администраторам!

Возможности:

  • Создание пользователя: введите ФИО — логин сгенерируется автоматически по ГОСТ Р 7.0.34-2014
  • Редактирование: изменение ФИО, роли, пароля
  • Просмотр ПК: кнопка "ПК" показывает закреплённое оборудование и позволяет добавлять/забирать компьютеры
  • Удаление: удаление пользователя (только если за ним не закреплены компьютеры)

Роли пользователей:

Роль Права
Администратор Полный доступ ко всем функциям, включая управление пользователями
Редактор Загрузка отчётов, обработка, передача компьютеров
Наблюдатель Только просмотр данных
Заблокирован Нет доступа к системе
Автогенерация логина: Если логин уже существует, система автоматически добавит дополнительные буквы из имени/отчества. Например: ivanov.ii → ivanov.iiи → ivanov.ivan и т.д.

13. Административные скрипты

В папке admin-scripts/ находятся два вспомогательных скрипта для администраторов:

🔧 fix-permissions.sh — Исправление прав доступа

Используется после каждого обновления через git pull для восстановления правильных прав на файлы.

Использование:

cd /volume1/web/inventory-web
./admin-scripts/fix-permissions.sh

Параметры (опционально):

./admin-scripts/fix-permissions.sh [ПУТЬ_К_ПРОЕКТУ] [ВЛАДЕЛЕЦ] [ГРУППА] [WEB_ПОЛЬЗОВАТЕЛЬ]

Пример с кастомными параметрами:

./admin-scripts/fix-permissions.sh /volume1/web/inventory-web myuser mygroup www-data
💡 По умолчанию: скрипт использует текущего пользователя и группу, а также пользователя http для веб-сервера.

🗑️ reset-inventory.sh — Сброс данных

Используется для очистки базы данных от тестовых данных. Удаляет все компьютеры, отчёты и передачи, но сохраняет пользователей.

Использование:

./admin-scripts/reset-inventory.sh

Параметры (опционально):

./admin-scripts/reset-inventory.sh [ПУТЬ_К_ПРОЕКТУ]
️ Осторожно: Скрипт запросит подтверждение перед удалением данных.

14. Экспорт в Excel

Функция экспорта доступна в разделах:

  • Компьютеры — полный реестр оборудования
  • История — журнал всех передач

Как экспортировать:

1 Перейдите в нужный раздел
2 Нажмите зелёную кнопку "Экспорт в Excel"
3 Файл автоматически скачается в формате .xls
4 Откройте файл в Microsoft Excel, LibreOffice Calc или другом табличном редакторе
✅ Совместимость: Файлы корректно отображают кириллицу и могут быть сохранены в формат .xlsx через сам Excel.
