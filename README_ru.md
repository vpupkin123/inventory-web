# 📋 Inventory Web

Лёгкое веб-приложение для учёта компьютерного оборудования в организациях. Оптимизировано для Synology NAS без внешних зависимостей.

## О проекте

**Inventory Web** решает задачу управления ИТ-активами путём автоматизации приёма данных об оборудовании. Клиентские машины генерируют JSON-отчёты с полной информацией о железе, которые загружаются в систему. Приложение парсит данные, фильтрует нерелевантные устройства (например, USB-накопители) и помещает компьютеры в очередь на обработку администратором.

Администраторы могут просматривать новое оборудование, назначать компьютеры сотрудникам, автоматически создавать учётные записи со стандартизированными логинами и вести полную историю всех перемещений техники. Система спроектирована так, чтобы быть простой, надёжной и лёгкой в обслуживании.

## Ключевые возможности

#### Автоматический приём

Парсинг JSON-отчётов об оборудовании с автоматической фильтрацией USB и съёмных накопителей.

#### 👤 Умное создание пользователей

Автоматическая генерация логинов из ФИО по стандарту транслитерации ГОСТ Р 7.0.34-2014 с встроенной обработкой совпадений.

#### 📋 Очередь обработки

Специальный интерфейс для проверки нового оборудования, добавления комментариев и настройки новых пользователей перед финальным назначением.

#### 🔄 Отслеживание активов

Закрепление компьютеров за конкретными сотрудниками или хранение на виртуальном складе с полной историей передач.

#### 📊 Экспорт в Excel

Экспорт реестра компьютеров и истории передач в один клик для формирования отчётности.

#### 🌍 Многоязычность

Полная поддержка русского и английского интерфейсов с удобным переключением.

## Структура репозитория

Проект имеет чистую архитектуру, похожую на MVC, с чётким разделением ответственности:

**admin-scripts/** — Скрипты для обслуживания сервера и управления правами доступа  
**config/** — Константы приложения, пути к базе данных и правила маршрутизации URL  
**controllers/** — PHP-классы, обрабатывающие логику каждой страницы (авторизация, дашборд, отчёты, обработка, компьютеры, пользователи и т.д.)  
**core/** — Движок приложения: собственный роутер, обработчик БД, аутентификация, автозагрузчик и интернационализация  
**data/** — Папка с данными времени выполнения: база данных SQLite, логи ошибок и загруженные JSON-отчёты  
**docs/** — HTML-файлы документации для пользователей и администраторов (на английском и русском)  
**lang/** — Файлы переводов для английского и русского интерфейсов  
**public/** — Корневая папка веб-сервера с единой точкой входа и правилами перезаписи URL  
**services/** — Вспомогательные классы, включая парсер JSON и движок транслитерации по ГОСТ  
**views/** — HTML-шаблоны для всех страниц, организованные по функциональным модулям

## Административные скрипты

Задачи регулярного обслуживания выполняются с помощью shell-скриптов в папке **admin-scripts/**. Эти скрипты универсальны и могут быть настроены через аргументы командной строки или переменные окружения.

### Доступные скрипты

*   **fix-permissions.sh** — Восстанавливает корректное владение файлами и папками, а также права доступа. Обязательно к запуску после получения обновлений из репозитория, чтобы веб-сервер мог записывать данные в базу и папку загрузок.
*   **reset-inventory.sh** — Очищает базу данных от всех тестовых данных (компьютеры, отчёты и история передач), сохраняя при этом учётные записи пользователей. Полезно для сброса окружения на этапах тестирования.
